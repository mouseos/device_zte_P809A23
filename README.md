# device_zte_P809A23 — CyanogenMod 12.1 device tree for ZTE ZTF32

CyanogenMod 12.1 (Android 5.1.1) device tree for the **ZTE ZTF32 / P809A23**
(KDDI-JP, Qualcomm MSM8909, Cortex-A7, Adreno 304, 512 MB RAM, 240x320 QVGA
ST7789V-SPI panel, Synaptics DSX touch).

> Status: device tree authored, **not yet built**. LineageOS has no 5.1 branch;
> the Android 5.1 line is CyanogenMod 12.1 (the LineageOS ancestor).

## Repos
- device tree: `device/zte/P809A23` (this repo)
- kernel: `kernel/zte/msm8909` — https://github.com/mouseos/kernel_zte_msm8909
  (ZTE OSS kernel 3.10.49; defconfig `msm8909-orion-perf_defconfig`,
  board DTS `msm8909-pm8916-zte-orion.dts`)
- vendor blobs: `vendor/zte/P809A23` — generated by `extract-files.sh` (proprietary,
  not committed). 297 blobs listed in `proprietary-files.txt`.

## Build (later)
```bash
# 1. CyanogenMod 12.1 source
repo init -u https://github.com/CyanogenMod/android.git -b cm-12.1
repo sync

# 2. place trees
git clone https://github.com/mouseos/device_zte_P809A23 device/zte/P809A23
git clone https://github.com/mouseos/kernel_zte_msm8909  kernel/zte/msm8909

# 3. blobs (device connected via adb, rooted)
./device/zte/P809A23/extract-files.sh

# 4. build
. build/envsetup.sh
brunch cm_P809A23
```

## Key facts baked in
- Kernel built from source (QCDT via `BOARD_KERNEL_SEPARATED_DT` + dtbToolCM;
  this aboot needs the QCDT DTB — see the TWRP repo for background).
- base 0x80000000, pagesize 2048, kernel_off 0x00008000, ramdisk_off 0x01000000,
  tags_off 0x00000100; boot/recovery 48 MB.
- Non-treble: vendor blobs live under /system/vendor.

## Tunables / TODO (verify at build/boot time)
- 240x320 framebuffer format / density (mdpi @ 140), low-RAM (512 MB) tuning.
- proprietary-files.txt is a first cut from the stock /system; prune AOSP-buildable
  libs if the build complains.
- sepolicy is a stub; expect to add device contexts.
